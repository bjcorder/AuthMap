# Code of Conduct

## Our standard

AuthMap is a defensive product-security project. We want collaboration here to
be direct, practical, and respectful, especially when discussing security
findings, false positives, and defensive boundaries.

Participants are expected to:

- use welcoming and inclusive language
- respect different levels of security and systems experience
- focus critique on ideas, code, and evidence
- avoid harassment, intimidation, personal attacks, and discriminatory conduct
- avoid posting exploit payloads, customer data, credentials, or sensitive
  vulnerability details in public project spaces

## Scope

This code of conduct applies in project repositories, issues, pull requests,
discussions, community spaces, and maintainer interactions related to AuthMap.

## Enforcement

Maintainers may edit, hide, or remove comments, issues, pull requests, or other
contributions that violate this code of conduct or the project's defensive-use
boundary. Maintainers may also temporarily or permanently restrict participation
for repeated or severe violations.

To report a concern, contact the repository owner privately. Do not open public
issues for harassment reports, private vulnerability details, credentials, or
customer-sensitive material.
