# Support

AuthMap is early-stage open-source software. Support is best-effort and centered
on defensive, authorized use.

## Where to ask

- Use GitHub Issues for bugs, feature requests, documentation gaps, and
  reproducible false-positive or false-negative reports.
- Use private security reporting for suspected vulnerabilities in AuthMap
  itself.
- Do not post credentials, customer code, exploit payloads, or sensitive
  vulnerability details in public issues.

## What to include

For bug reports, include:

- AuthMap version or commit SHA
- operating system and installation method
- command run and sanitized output
- minimal sanitized fixture or code shape, when possible
- expected behavior and actual behavior

For detection improvements, include:

- framework and language
- sanitized route or guard pattern
- expected evidence type or coverage class
- why the pattern is server-side authorization evidence

## Support boundaries

AuthMap maintainers will not help with exploit automation, unauthorized
scanning, credential theft, bypass instructions, or attacks against live
systems.
